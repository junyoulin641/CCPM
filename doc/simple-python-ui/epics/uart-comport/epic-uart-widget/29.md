# Task: Add error message display UI

## Description
Add user-visible error message display to the UARTWidget UI. When connection errors, operation failures, or device issues occur, display clear error messages to the user. This can be implemented as a persistent QLabel for status messages or QMessageBox for critical errors. Messages should be cleared on successful operations.

## Acceptance Criteria
- [ ] Error display UI element added (QLabel or QMessageBox)
- [ ] errorOccurred signal connected to display handler
- [ ] Connection errors displayed ("Failed to open port COM3")
- [ ] Disconnect errors displayed ("Device disconnected unexpectedly")
- [ ] Error messages cleared on successful connection
- [ ] Error text is readable and user-friendly
- [ ] Error display doesn't obscure other UI elements

## Technical Details
**Implementation approach:**
Option 1 - Status Label (recommended):
- Add QLabel at bottom of widget for status messages
- Style with red text for errors, green for success
- Connect errorOccurred signal to update label
- Auto-clear on next successful operation

Option 2 - Message Box:
- Use QMessageBox.warning() for critical errors
- May be disruptive for frequent errors
- Better for one-time configuration errors

**Key considerations:**
- Status label approach is less disruptive
- Error messages should be specific but not overly technical
- Include error context (port name, baud rate, etc.)
- Consider adding timestamp to status messages

**Code locations/files affected:**
- `uart_widget.py`: Add QLabel to layout in _create_ui()
- `uart_widget.py`: Add error display handler method
- `uart_widget.py`: Connect errorOccurred signal to handler
